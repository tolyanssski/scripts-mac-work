# Правки проекта br-users, пакет notifier: реализовать ограничение количества отправляемых доставок уведомлений по каналам sms и confirmation_code

В методы нотифаера, занимающиеся доставкой уведомления по каналам смс и confirmation_code
добавь проверку на то, сколько доставок уведомлений по этим каналам для заданного пользователя
уже успешно завершено за последние N минут/часов и тд (конкретный интервал времени и сколько максимально разрешать сообщений - задай в конфиге).
Если уже было за настроенный в конфиге период времени более чем максимально разрешенное количество доставок по этому каналу для этого юзера, доставку просто переводи в статус
skipped_by_rate_limit, сохраняй в БД и выходи из метода-обработчика без ошибок, не выполняя никаких отправок никуда.
